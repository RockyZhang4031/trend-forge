# Trend Forge — 产品意图与交互规格书

> **本文档用途：** 供 AI 理解我们在做什么、为什么做、做到什么程度。  
> 任何 AI 拿到此文档，应能理解产品的完整愿景、当前能力边界、以及前端交互的进化方向。

---

## 一、产品是什么

**Trend Forge（趋势锻造）** 是一个**人类发展进程可视化推演平台**。

不是新闻聚合器，不是搜索引擎，不是数据看板。  
它做的事是：**把人类文明发展中"必然发生的趋势"的因果链画出来**，让用户像看星图一样看到——

- 什么技术在驱动什么产业
- 什么资源在制约什么发展
- 什么人物在推动什么变革
- 节点之间如何相互依赖、竞争、引发

**核心隐喻：** 一张不断生长的"因果星图"。用户是观测者也是锻造者——可以喂数据让图生长，可以评论标记认知，可以截图分享观点。

## 二、产品在做什么事

### 2.1 数据流

```
用户喂数据（文本/新闻/论文/报告）
    ↓
AI 提取实体（节点）和关系（连线）
    ↓
图谱生长（新节点从天空坠落入画布，新连线像激光射出）
    ↓
用户观测、评论、截图、分享
    ↓
快照记录图谱演变历史
```

### 2.2 核心循环

1. **看图** — 浏览趋势图谱，点击节点看详情
2. **喂养** — 粘贴文本，AI 提取实体，图谱生长
3. **标注** — 对节点/主题发表评论
4. **推演** — 切换时间快照，看图谱如何演变
5. **分享** — 截图导出，传播观点

## 三、数据模型（已有）

### 主题 (Theme)
| 字段 | 类型 | 说明 |
|------|------|------|
| name | string | 主题名称（如"机器人发展探索"） |
| description | text | 主题描述 |
| heat_score | 0-100 | 主题热度 |
| status | 0/1/2 | 0=公开 1=付费 2=私有 |

### 节点 (Node) — 图谱中的"星体"
| 字段 | 类型 | 说明 |
|------|------|------|
| name | string | 节点名称 |
| type | 1-7 | 1趋势 2技术 3资源 4公司 5人物 6产业 7事件 |
| scarcity_score | 0-100 | 稀缺度（越高越稀缺） |
| influence_score | 0-100 | 影响力（决定节点大小/亮度） |
| confidence_score | 0-100 | 置信度（数据可信程度） |
| lifecycle_stage | 0-100 | 生命周期阶段（0萌芽→100衰退） |
| lifecycle_pos | 0-100 | 当前在生命周期中的位置 |
| market_size | float | 市场规模（亿元） |
| growth_rate | float | 增长率（%） |
| metadata | jsonb | 扩展字段 |
| x, y | float | 画布坐标 |

### 关系 (Edge) — 星体间的"引力束"
| 字段 | 类型 | 说明 |
|------|------|------|
| source → target | uuid | 有向关系 |
| type | 1-5 | 1依赖 2推动 3掌控 4属于 5引发 |
| weight | 0-100 | 关系强度（决定连线粗细） |
| confidence_score | 0-100 | 置信度 |

### 喂养记录 (Feed)
| 字段 | 类型 | 说明 |
|------|------|------|
| source_type | 1-4 | 1新闻 2论文 3用户输入 4报告 |
| raw_content | text | 原始文本 |
| status | 0-3 | 0待处理 1处理中 2完成 3失败 |
| result_summary | text | AI 处理摘要 |

### 快照 (Snapshot)
| 字段 | 类型 | 说明 |
|------|------|------|
| node_count | int | 节点数 |
| edge_count | int | 关系数 |
| diff_data | jsonb | 增量数据（相比上次快照新增了什么） |

### 评论 (Comment)
| 字段 | 类型 | 说明 |
|------|------|------|
| theme_id | uuid | 主题级评论 |
| node_id | uuid | 节点级评论 |
| parent_id | uuid | 支持回复 |
| author_name | string | 昵称 |

### 用户 (User — 预留)
| 字段 | 类型 | 说明 |
|------|------|------|
| role | 0-3 | 0访客 1用户 2付费 3管理员 |

## 六、设计愿景与进化方向

### 6.1 终极愿景

> **让用户感觉自己在操作一个"深空数据终端"——不是在看图表，而是在驾驭文明发展的因果星图。**

参考体验：
- **Palantir Gotham** 的冷峻专业感 — 数据密集但不混乱
- **Apple Vision Pro** 的空间深度 — 元素悬浮于不同 Z 轴层级
- **Cyberpunk 2077** 的霓虹数据流 — 算力感、未来感
- **Stellaris/星际争霸** 的星图界面 — 像在俯瞰一个文明

### 6.2 交互进化路线图

**Phase 1（当前 ✅）：3D 全息骨架**
- Three.js + R3F 渲染
- 玻璃节点 + 线框旋转 + 呼吸灯
- 流动光点连线
- 粒子星云背景
- Fly-To 相机聚焦

**Phase 2：后处理 + 材质升级**
- UnrealBloomPass 辉光后处理
- 节点物理材质（金属/玻璃/能量场）
- 自定义 Shader 连线（能量流动纹理）
- 景深后处理（远处模糊）
- 节点爆炸/合并粒子特效

**Phase 3：时间轴推演**
- 底部时间轴滑块
- 快照 diff 动画（节点从天空坠入、连线激光射出）
- 关键事件标记（节点闪烁警报色）
- 自动播放模式（图谱随时间自动生长）

**Phase 4：沉浸式虚拟世界**
- 第一人称漫游模式（PointerLockControls）
- 节点如星球，连线如星际航线
- 3D 全息信息板（节点上方悬浮）
- 音画同步（粒子运动 + 音频频谱）
- 多人协作（WebSocket 实时同步）

**Phase 5：AI 自主演化**
- AI 自动监控数据源 → 自动喂养 → 图谱自动生长
- AI 推演未来趋势 → 生成"预测节点"（虚影显示）
- AI 因果链推理 → 自动补全缺失关系

### 6.3 视觉设计原则

1. **每一个像素都要传递"算力"和"推演"的质感**
2. **拒绝扁平化** — 所有元素有深度、有材质、有光影
3. **动态优先** — 数据变化时有动画反馈（节点坠落、连线射出、数字翻滚）
4. **空间感优先** — Z 轴深度映射，近大远小，近清远模糊
5. **物理弹性** — 所有动画用 spring 物理，不用线性
6. **暗色深空** — 主背景 #05070A，所有元素发光浮于其上
7. **电青主色** — #00F0FF 作为主交互色，语义辅助色区分类型

### 6.4 色彩语义系统

| 颜色 | 色值 | 语义 |
|------|------|------|
| 电青 | #00F0FF | 主品牌色、技术节点、高亮、主连线 |
| 暖橙 | #FF6B35 | 热力/预警、人物节点、高热度 |
| 冷紫 | #6C5CE7 | 算力/AI、公司节点、依赖关系 |
| 青绿 | #00D9A5 | 增长/正向、概念节点、推动关系 |
| 警示红 | #FF2E63 | 风险/关键、竞争关系、警报 |
| 金色 | #FFD700 | 资源节点、稀缺材料 |
| 深空黑 | #05070A | 主背景 |
| 面板蓝 | rgba(10,14,23,0.75) | 玻璃面板背景 |
