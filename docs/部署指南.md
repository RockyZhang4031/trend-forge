# Trend Forge - 部署指南

## 前置条件

你需要注册以下免费账号：

| 平台 | 用途 | 注册地址 |
|---|---|---|
| GitHub | 代码仓库 | https://github.com |
| Vercel | 前端托管 | https://vercel.com |
| Supabase | 数据库 | https://supabase.com |
| Render | 后端托管 | https://render.com |
| 智谱开放平台 | AI API | https://open.bigmodel.cn |

---

## 步骤一：Supabase 数据库配置

1. 登录 Supabase → 创建新项目
2. 项目名：`trend-forge`
3. 设置数据库密码（记下来）
4. 等待项目初始化完成
5. 进入 SQL Editor
6. 复制 `backend/migrations/001_init.sql` 全部内容，粘贴执行
7. 进入 Settings → API
8. 记下以下信息：
   - Project URL → `SUPABASE_URL`
   - anon public key → `SUPABASE_ANON_KEY`
   - service_role key → `SUPABASE_SERVICE_KEY`

---

## 步骤二：智谱 GLM API 配置

1. 登录智谱开放平台
2. 进入 API Keys → 创建新 Key
3. 记下 Key → `GLM_API_KEY`
4. 免费额度：注册送代金券，glm-4-flash 模型免费

---

## 步骤三：本地开发配置

### 后端

```bash
cd trend-forge/backend
cp .env.example .env
# 编辑 .env，填入 Supabase 和 GLM 的信息
npm install
npm run seed   # 生成种子数据
npm run dev    # 启动后端 http://localhost:3001
```

### 前端

```bash
cd trend-forge/frontend
npm install
# 创建 .env.local
echo "VITE_API_URL=http://localhost:3001" > .env.local
npm run dev    # 启动前端 http://localhost:5173
```

打开 http://localhost:5173 即可看到效果。

## 步骤四：部署上线

### 4.1 推送到 GitHub

```bash
cd trend-forge
git init
git add .
git commit -m "Initial commit: Trend Forge"
# 在 GitHub 创建仓库 trend-forge
git remote add origin https://github.com/你的用户名/trend-forge.git
git push -u origin main
```

### 4.2 部署后端到 Render

1. 登录 Render → New → Web Service
2. 连接 GitHub 仓库
3. 配置：
   - Name: `trend-forge-api`
   - Root Directory: `backend`
   - Build Command: `npm install`
   - Start Command: `npm start`
4. 环境变量（在 Render 后台填入）：
   - `SUPABASE_URL` = 你的值
   - `SUPABASE_ANON_KEY` = 你的值
   - `SUPABASE_SERVICE_KEY` = 你的值
   - `AI_PROVIDER` = `glm`
   - `GLM_API_KEY` = 你的值
   - `GLM_API_URL` = `https://open.bigmodel.cn/api/paas/v4`
   - `AI_MODEL` = `glm-4-flash`
   - `NODE_ENV` = `production`
   - `CORS_ORIGIN` = `https://trend-forge.vercel.app`（部署完前端后回来填）
5. 部署完成后记下地址：`https://trend-forge-api.onrender.com`

### 4.3 部署前端到 Vercel

1. 登录 Vercel → New Project
2. Import GitHub 仓库
3. 配置：
   - Root Directory: `frontend`
   - Framework Preset: Vite
   - Build Command: `npm run build`
   - Output Directory: `dist`
4. 环境变量：
   - `VITE_API_URL` = `https://trend-forge-api.onrender.com`
5. Deploy
6. 部署完成后地址：`https://trend-forge.vercel.app`

### 4.4 回填 CORS

回到 Render → trend-forge-api → Environment：
- 将 `CORS_ORIGIN` 更新为你的 Vercel 域名
- 保存后会自动重新部署

## 环境变量汇总

| 变量 | 在哪用 | 说明 |
|---|---|---|
| `SUPABASE_URL` | 后端 | Supabase 项目 URL |
| `SUPABASE_ANON_KEY` | 后端 | Supabase 匿名 Key |
| `SUPABASE_SERVICE_KEY` | 后端 | Supabase 服务 Key |
| `AI_PROVIDER` | 后端 | AI 提供商，当前 `glm` |
| `GLM_API_KEY` | 后端 | 智谱 API Key |
| `GLM_API_URL` | 后端 | 智谱 API 地址 |
| `AI_MODEL` | 后端 | 模型名称 |
| `CORS_ORIGIN` | 后端 | 前端域名 |
| `VITE_API_URL` | 前端 | 后端 API 地址 |
