【一】项目定位与设计愿景

1.1 核心命题

Trend Forge 不是"数据可视化工具"，而是「人类文明的时空推演操作系统」。

用户不是"查看"知识图谱，而是"驾驶"一艘穿梭于历史-现在-未来的探测舰，
在三维时空连续体中追踪因果链、模拟分支未来、发现隐藏关联。

1.2 设计愿景：「深空考古终端」

视觉锚点：Palantir Gotham 的冷峻专业感 × Cyberpunk 2077 的数据霓虹
× Apple Vision Pro 的空间深度 × 星际穿越中的四维时空可视化

交互锚点：从"点击-查看"的平面交互，进化为"飞行-探索-推演"的沉浸式
          空间交互。每一次操作都应产生物理反馈，每一次切换都应伴随
          时空跃迁的仪式感。

1.3 设计原则（5大铁律）

┌─────────────────────────────────────────────────────────────────────┐
│ ① 空间优先：所有信息必须存在于三维空间的不同深度层级，拒绝平面堆砌   │
│ ② 物理真实：交互必须有质量、有惯性、有弹性，拒绝线性动画              │
│ ③ 语义发光：颜色不只是装饰，而是数据语义的直接表达（热度=亮度=高度）  │
│ ④ 即时推演：任何操作都应在 100ms 内产生视觉反馈，拒绝等待焦虑         │
│ ⑤ 渐进披露：信息按探索深度分层呈现，拒绝一次性信息轰炸                │
└─────────────────────────────────────────────────────────────────────┘

3.2 节点渲染系统：「时空奇点」

3.2.1 节点几何形态（按类型区分）

┌──────────────┬──────────────────────────────────────────────────────┐
│ 类型         │ 视觉形态                                              │
├──────────────┼──────────────────────────────────────────────────────┤
│ 事件/节点    │ 多面晶体球（Icosahedron）+ 玻璃材质 + 内部缓慢旋转    │
│ 人物         │ 发光球体 + 脉冲呼吸环 + 头像投影（Billboard）         │
│ 概念/理论    │ 正八面体 + 线框覆盖 + 顶点发光                        │
│ 技术/发明    │ 圆柱体 + 能量核心（内部点光源）+ 表面流光              │
│ 资源/材料    │ 不规则陨石形态 + 金属粗糙度 + 环境反射                  │
│ 机构/国家    │ 圆角立方体 + 徽标浮雕 + 扫描线动画                    │
└──────────────┴──────────────────────────────────────────────────────┘

3.2.2 节点材质规范（MeshPhysicalMaterial）

┌─────────────────────────────────────────────────────────────────────┐
│ color:        基于类型的语义色（见第6章设计令牌）                    │
│ transparent:  true                                                   │
│ opacity:      0.75 ~ 0.95（选中时1.0，非焦点时0.3）                 │
│ roughness:    0.1（玻璃感）                                          │
│ metalness:    0.2                                                    │
│ transmission: 0.5（透光率）                                          │
│ thickness:    1.5                                                    │
│ emissive:     同 color                                               │
│ emissiveIntensity: 0.3（基础）~ 1.2（选中/高热度）                  │
│ clearcoat:    1.0                                                    │
│ clearcoatRoughness: 0.1                                              │
└─────────────────────────────────────────────────────────────────────┘

3.2.3 节点动态行为

• 呼吸动画：所有节点持续进行正弦波缩放（周期 3~5s，相位随机偏移）
• 布朗微动：平衡位置附近±0.3单位的随机漂移（模拟量子不确定性）
• 热度映射：heat值越高，节点悬浮高度（Z轴）越靠近用户，发光越强
• 选中态：外环扩张消失（冲击波）+ 内部几何体旋转加速×3 + 相机聚焦

3.3 连线渲染系统：「因果光束」

3.3.1 连线视觉规范

• 基础形态：使用 THREE.Line2（可设置线宽）或自定义 Shader 管道
• 颜色编码：
  - 因果关系（causes）    → 电青色 #00F0FF，流动方向：因→果
  - 影响关系（influences）→ 琥珀色 #FF6B35，双向流动
  - 从属关系（belongs）   → 银灰色 #8B95A5，静态微光
  - 对抗关系（competes）   → 警报红 #FF2E63，闪烁警示
• 粗细映射：strength 0~1 → lineWidth 0.5~4.0
• 流动效果：连线中心有高光粒子以 speed = strength × 2 的速度流动

3.3.2 连线 Shader 核心逻辑（伪代码）

┌─────────────────────────────────────────────────────────────────────┐
│ uniform float uTime;          // 时间 uniforms                       │
│ uniform float uSpeed;         // 流动速度                           │
│ uniform vec3  uColor;         // 基础颜色                           │
│                                                                     │
│ void main() {                                                       │
│   float flow = fract(vUv.x - uTime * uSpeed);                      │
│   float glow = smoothstep(0.0, 0.15, flow) * smoothstep(0.3, 0.15, flow);│
│   vec3 finalColor = mix(uColor * 0.3, uColor * 2.0, glow);         │
│   gl_FragColor = vec4(finalColor, vAlpha);                         │
│ }                                                                   │
└─────────────────────────────────────────────────────────────────────┘

3.4 粒子氛围系统：「时空尘埃」

• 数量：1500~3000 点（根据设备性能自适应）
• 材质：PointsMaterial + AdditiveBlending + sizeAttenuation
• 行为：
  - 基础：缓慢布朗运动（速度 0.01~0.03）
  - 交互：鼠标移动产生排斥场（半径 40，力强度 0.1）
  - 节点引力：粒子被高热度节点吸引，形成微弱星环
  - 时间轴：推演播放时，粒子流动方向与"时间箭头"一致
• 颜色：主色调电青色 #00F0FF，透明度 0.3~0.6

3.5 后处理效果栈（PostProcessing）

渲染顺序（必须按此顺序）：
① SSAO（环境光遮蔽）     → 增强节点与地面接触的真实感
② Bloom（辉光）          → 阈值 0.8，强度 1.5，半径 0.5
③ DepthOfField（景深）   → 聚焦距离跟随选中节点，模糊半径 0.02
④ MotionBlur（运动模糊） → 仅在相机快速移动或拖拽时启用
⑤ Vignette（暗角）       → 强度 0.3，边缘压暗引导视觉中心
⑥ ToneMapping（色调映射）→ ACESFilmic，曝光 1.2

【四】游戏化交互系统：「文明推演」玩法设计

4.1 核心玩法循环（Gameplay Loop）

┌─────────────────────────────────────────────────────────────────────┐
│                                                                     │
│    探索（Explore） → 发现（Discover） → 推演（Simulate）            │
│         ↑                                              ↓            │
│         └──────────── 分享/协作（Collaborate） ←───────┘            │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘

4.1.1 探索阶段（Explore）

• 用户像驾驶飞船一样在三维空间中自由航行（WASD/拖拽/滚轮）
• 接近节点时，雷达扫描音效 + 节点脉冲加速
• 发现隐藏关联时，触发「发现」动画（粒子爆发 + 音效 + 经验值+10）

4.1.2 推演阶段（Simulate）

• 用户选择关键节点作为「推演起点」，系统生成「平行未来」分支
• 时间轴滑块控制推演进度，节点和连线随时间动态变化
• 关键事件发生时，全屏震动 + 红色警报脉冲 + 自动弹出事件卡

4.1.3 协作阶段（Collaborate）

• 用户可保存当前视角为「时空快照」（Snapshot）
• 快照可分享至社区，其他用户可「 fork 」并继续推演
• 社区热度高的推演路径显示为「高亮航线」

4.3 时间轴推演控制器：「跃迁引擎」

视觉设计：
• 底部中央悬浮一个「弧形时间滑轨」，像星际飞船的油门推杆
• 滑轨上有发光刻度标记历史纪元（古代/近代/现代/未来）
• 当前时间指针是一个发光的三角形，拖拽时有拖尾光效

交互逻辑：
• 播放按钮：节点和连线随时间轴动态变化（新增/消失/变色）
• 倍速控制：1x / 5x / 10x / 50x / 100x
• 关键帧标记：用户可在任意时间点打标记，快速跳转
• 分支预览：悬停未来时间点，显示「概率云」（半透明节点预览）

spring(stiffness: 300~500, damping: 20~35, mass: 0.8~1.2)

7.3 性能优化策略

┌─────────────────────────────────────────────────────────────────────┐
│ 渲染优化                                                             │
│   • 节点 > 100：使用 InstancedMesh 批量渲染同类几何体               │
│   • 粒子：单 BufferGeometry + Points，1 draw call                   │
│   • 连线：LineSegments 合并渲染，减少 draw call                     │
│   • LOD：远距离节点降级为简单球体 + 隐藏标签                        │
│   • Frustum Culling：视锥体外节点/连线不渲染                      │
│                                                                     │
│ 内存优化                                                             │
│   • 纹理压缩：使用 KTX2/Basis Universal 格式                        │
│   • 几何体复用：同类节点共享 BufferGeometry                         │
│   • 对象池：粒子/特效对象复用，避免频繁创建/销毁                    │
│                                                                     │
│ 加载优化                                                             │
│   • 代码分割：3D 场景懒加载，首屏只加载 UI 框架                     │
│   • 数据分页：节点列表虚拟滚动，图谱数据按需加载                    │
│   • 骨架屏：3D 场景加载时显示星空加载动画                           │
│                                                                     │
│ 响应式降级                                                           │
│   • 高性能设备：全开（Bloom + SSAO + 景深 + 3000 粒子）             │
│   • 中性能设备：半开（Bloom + 1500 粒子，关闭 SSAO）                │
│   • 低性能设备：基础（无后期处理，500 粒子，简化材质）              │

8.2 差异化竞争优势

当前市面上的知识图谱工具（如 PuppyGraph、Neo4j Bloom、Kibana Graph）
普遍停留在「2D 平面 + 表格面板」的阶段。Trend Forge 通过本设计方案
将获得以下差异化优势：

① 空间认知优势：三维布局让大脑自然理解复杂网络的层级和聚类
② 情感连接优势：游戏化成就系统和物理反馈创造「探索的愉悦感」
③ 推演沉浸优势：时间轴动画让「历史进程」从静态图表变为动态叙事
④ AI 融合优势：AI 不是侧边栏的聊天框，而是空间中可见的「紫色光环」
⑤ 社区叙事优势：用户创造的推演路径成为可视化的「文明航线图」
